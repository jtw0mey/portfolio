    For this project I will be using Power BI's DAX language to create a sales report for a fictional football team, "Oslo FC". I will be displaying merchandise sales data from four sports stores that stock Oslo FC merchandise.

    The dataset contains two tables. The first table contains each item of Oslo FC merchandise with its price and production cost:









    The second table contains sales data of what merchandise each store has sold. A subset of the table is here:

    I will start by uploading the dataset to Power BI. It has automatically recognised that there is a relationship between the two tables, and they have been joined together using the mutual Product ID  column found in both tables.









    I started by creating a Count of Sales  column to show how many individual merchandise orders each store is making using the COUNT  function. I was then able to create the table and bar chart below.









    I then used the formula below to create a Products Sold  column. This was done by totaling the Units Sold  using the SUM  function. This combined the Product Name from the Football Store table, and the Products Sold from the Football Sales table using the joint product ID.









    I then wanted to create a Profit Column  for each merchandise item. I first tried a simple equation of Price minus Product cost, multiplied by Units Sold. This was the wrong equation as it produced the same result for each row. This was because the equation was multiplied by the sum off all Units Sold, rather than multiplied by the Units Sold for each individual product.









    This Profit Column  was corrected by using SUMX  to run this equation on each individual row, rather than as one total price. We also use RELATEDTABLE  to access all the data from the Football Sales Table to be able to create this Profit Column in the Football Store Table.









    Using SUMX  again, I was also able to create the table and bar chart below for Profit by Customer.









    Next, I wanted to create a Weekday  column to be able to visualize sales by day of the week. I used the Weekday function to transform the Date Purchased column in to the number representing the day of the week. 1 = Monday, 2 = Tuesday etc. I was then able to create the table and bar chart below which showed us how Saturday is the busiest day for sales.









    I then wanted to create a column that designates each sale to either a Big Order  or a Small Order  based on whether the sale had over/under 30 units sold. For this I used an IF  function as seen below.
